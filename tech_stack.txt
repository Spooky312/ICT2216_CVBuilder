================================================================================
  SECURITY LIBRARIES
================================================================================

  Package / Feature     Version   Description
  --------------------  --------  -----------------------------------------------
  bcrypt                4.1.3     Adaptive password hashing (bcrypt algorithm)
                                  with configurable cost factor
  cryptography          42.0.8    Low-level cryptographic primitives — used
                                  internally by other security dependencies
  Flask-JWT-Extended    4.6.0     Manages JWT lifecycle; enforces HttpOnly
                                  cookie storage, SameSite=Strict, and per-
                                  request CSRF token validation
  Flask-Limiter         3.7.0     Brute-force protection — locks out IPs and
                                  accounts after repeated failed login attempts
  secrets (stdlib)      built-in  Cryptographically secure random token
                                  generation (Python standard library)
  hashlib (stdlib)      built-in  SHA-256 hashing of tokens before DB storage
                                  (Python standard library)

  Security controls implemented in application code (no extra package):
  - Content Security Policy (CSP) headers on every response
  - X-Frame-Options: DENY
  - X-Content-Type-Options: nosniff
  - Referrer-Policy: strict-origin-when-cross-origin
  - Strict-Transport-Security (HSTS) when running with TLS
  - Account lockout after 5 consecutive failed logins (15-minute window)
  - Input validation and sanitisation via marshmallow schemas

================================================================================
  CI/CD TOOLS
================================================================================

  Tool                  Description
  --------------------  -------------------------------------------------------
  GitHub Actions        CI/CD platform — runs the pipeline defined in
                        .github/workflows/ci.yml on every push and pull request
  actions/checkout      Official GitHub Action to check out the repository
  actions/setup-python  Official GitHub Action to install Python 3.12
  actions/setup-node    Official GitHub Action to install Node.js 20 with
                        npm cache enabled
  safety                Python dependency vulnerability scanner — checks
                        requirements.txt against known CVE databases
  npm audit             Node.js built-in dependency vulnerability audit —
                        flags packages with known high-severity issues

  CI Pipeline Jobs:
    1. backend-test      Spins up PostgreSQL, installs Python deps, runs pytest
    2. frontend-build    Installs Node deps, runs npm run build (Vite)
    3. dependency-audit  Runs safety (Python) and npm audit (Node)

================================================================================
  INFRASTRUCTURE & OTHERS
================================================================================

  Tool / Technology     Version   Description
  --------------------  --------  -----------------------------------------------
  Docker                24+       Containerises backend, frontend, and Nginx
                                  into isolated, reproducible images
